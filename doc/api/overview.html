sierra-php v1.1.3 packages

Name Comment
etc This directory contains system wide configurations including the following:

app-config.dtd - dtd for specific application configurations (app-config.xml in app/[app name]/etc)

installer.dtd - the dtd for installer configurations

installer.xml - the sierra-php framework installer configuration

sierra-config.dtd - the dtd for validation and description of the sierra-php framework configuration

l10n - see l10n/README
etc.l10n This directory contains localization enabling files including xml files for describing address formats, currencies and locales as well as localizable properties files for system wide strings. the SRA_ResourceBundle returned by SRA_Controller::getAppResources will contain both the application as well as the platform resources defined in sierra.properties
auth The authentication classes can be used to secure areas of your application based on the authentication configuration provided in app-config. Usage of these classes are documented thoroughly in the app-config dtd
core Contains the core framework classes, SRA_Controller being the foremost of them all. SRA_Controller MUST be included in each PHP source file that utilizes the framework. For more information, see app/sra-app/www/html/index.php
model The SRA_EntityModeler allows SIERRA framework applications to define their object model in xml including entities, entity relationships, validation, and o/r mappings. The SRA_EntityModeler is essentially a DAO/VO generator. No build is required as all initializations are performed automatically the first time that the entity model is used and thereafter only updated when changes are made to the entity model.

For more information, see the app-config dtd and the entity-model dtd.
sql This directory/package contains the database related classes of the SIERRA PHP deployment framework. All of these classes are accessible without requiring includes ONCE THE SRA_Controller.php class has been included.

Most classes are fairly self explanatory based on the file name. For more information, please review the class comments or the SIERRA api documentation. SRA_Database configuration directives are specified in the sierra-config.xml and app-config.xml files. For more information, please review the sample xml configurations, and the accompanying dtds.
test SRA_TestCase is the base class for all unit tests. For more information, review the api in SRA_TestCase as well as sierra/bin/run-tests.php
util This directory/package contains the utility related classes of the SIERRA PHP deployment framework. For more information, please review the class comments or the SIERRA api documentation.
util.installer No package comment
util.l10n No package comment
workflow This package provides a means of incorporating declarative workflow logic into an application. For more information, review workflow.dtd and WorkflowManager.php.

SRA_VERSION_FILE


 the name of the version information file (stored in the framework and app
  "etc" directory). this file contains the following values:
    svn-ci-password=[overrides svn-password for checkin operations]
    svn-ci-path=[overrides svn-path for checkin operations]
    svn-ci-url=[overrides svn-url for checkin operations]
    svn-ci-user=[overrides svn-user for checkin operations]
    svn-password=[the subversion repository password]
    svn-path=[the name of the framework/app subversion repository]
    svn-url=[the url to the subversion repository containing the framework/app]
    svn-user=[the subversion repository username]
    version=[current version identifier]
    versions-info=[path or uri to the versions info file]
    versions-readme=[path or uri to the versions readme file]
  the versions-info file contains a list of all of the available versions in
  the format (where the topmost entry is considered the latest version):
    [version id]=[path or uri to source archive]
  the versions-readme file contains a list of the readme notes for each version
  in the format:
    [version id]=[path or uri to change info notes]
  use the SRA_Controller::getAppSvnInfo, SRA_Controller::getSysSvnInfo,
  SRA_Controller::getAppVersion, SRA_Controller::getSysVersion,
  SRA_Controller::getAppVersionRb, SRA_Controller::getSysVersionRb,
  SRA_Controller::getAppVersions, SRA_Controller::getSysVersions,
  SRA_Controller::getAppLatestVersion, and SRA_Controller::getSysLatestVersion
  methods to retrieve and use the data provided in these files
